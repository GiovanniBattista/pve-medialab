Kodirsync Installation Package

Your new Kodirsync account is active and ready to use.

Your connection settings have been pre-set by your Kodirsync server administrator. All connections are made using a private SSH key which is included in your installer package. If you are configured for remote access then you will connect using SSLH or Port Forward protocols.

  1. LAN Connection - Local connectivity

    LAN is the priority connection protocol whenever your Kodirsync client is connected on the same LAN network as the Kodirsync server.

  2. SSLH Connection - Remote connectivity

    HTTPS SSL 443 provides a additional layer of security for client connection authentication and encrypted data transport. SSLH requires the client to use SSL certificates and keys in addition to your standard SSH key credentials which are included in the installer package.

  3. Port Forward - Remote connectivity

    Port forwarding is a basic method to provide remote access to the Kodirsync server. Connection authentication is with SSH key only.

More information about Kodirsync is available on our GitHub page.


Installation Instructions

Installing is a simple procedure. The attached Installer package is preset with all your access credentials. Simply follow our instructions. If you are using USB based storage for Kodirsync always have your client device connected with a USB blank storage device (disk) before running our installer.

If you are new to SSH and use Windows OS we recommend Putty SSH software.

  1. Copy Installer to client device

    2. Copy the email attachment file 'installer.run' to the following folder location on your client device.

    1. CoreElec and LibreElec devices
      2. ~/backup
    2. Other Linux hardware
      3. /tmp

  2. Run the Kodirsync installer

    3. SSH into your client device as root (i.e ssh root@coreelec.local and default pwd is 'coreelec'). Type the following command (or copy & paste) including end brackets into your SSH terminal:

    1. CoreElec or LibreElec
      2. (mkdir -p /tmp/selftar ; cd /tmp/selftar ; ~/backup/installer.run)
    2. Other Linux hardware
      3. (mkdir -p /tmp/selftar ; cd /tmp/selftar ; /tmp/installer.run)

    The above command will run our Kodirsync device installer. Follow the prompts.


Uninstall & Remove Kodirsync

SSH into your client device as root. Type the following command (or copy & paste) in your SSH terminal and select the uninstall option when prompted.

  1. CoreElec or LibreElec
    2. (mkdir -p /tmp/selftar ; cd /tmp/selftar ; ~/backup/installer.run)
  2. Other Linux hardware
    3. (mkdir -p /tmp/selftar ; cd /tmp/selftar ; /tmp/installer.run)

Updating Kodirsync client SW

Kodirsync automatically updates its application scripts once a week to our latest GitHub version. Obviously your settings are maintained.

If you want to perform a manual forced upgrade to the latest version you must run a CLI command. SSH into your Kodirsync client device as root. Type the following command (or copy & paste) in your SSH terminal and press enter.

  1. CoreElec or LibreElec
    2. /storage/kodirsync/kodirsync_clientapp_gitupdater.sh
  2. Other Linux hardware
    3. /usr/local/bin/kodirsync/kodirsync_clientapp_gitupdater.sh

Manually performing a update will kill all running Kodirsync processes. The processes will restart at the next scheduled synchronization of media.


Manual editing of the User configuration file

We do not recommend Users make manual Kodirsync changes unless they know what they are doing. Remember, changing a users content source types (i.e movies, series, photos) can only be done at the Kodirsync server.

Users with basic Linux skills you can make configuration changes in the 'kodirsync_clientapp_user.cfg' file. Do NOT edit any other file. Other changes must be performed server-side. Best use nano or vi only.

  1. CoreElec and LibreElec
    2. nano /storage/kodirsync/kodirsync_clientapp_user.cfg
  2. Other Linux hardware
    3. nano /usr/local/bin/kodirsync/kodirsync_clientapp_user.cfg

Attached files: 'installer.run'
